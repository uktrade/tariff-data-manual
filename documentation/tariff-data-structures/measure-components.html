‘Measure components’ are the elements that make up a tariff duty.

Not all tariff measures have duties associated with them. For example, measures of type 103, 142 or 143 (third country duty, tariff preference and preferential tariff rate quota) require duties but, for example 277 import prohibition does not.

This table shows the duties or measure component column on the UK Trade Tariff service.

Types of duty applied to measures

The types of duties that can be applied to a measure are:

  • ‘Ad valorem’ duty, for example 0.00% or 3.60% (a percentage additional duty applied to the import duty of the goods
  • specific duty: an import duty dependent on the weight or volume of goods being imported rather than the price (for example 4.200 EUR/kg or 110.00 EUR/l)
  • compound duty: a combination of ad valorem and compound types (for example 3.60% + 110 EUR/l) ‘Conjunctions’ can be used to join multiple ‘clauses’ within the duty sentence. For example, Min (down to a minimum of …) Max (up to a maximum of …)

Measure components

A ‘measure component’ is the term used in tariff management to describe the elements that make up a tariff duty.

An example

The duties applied to the import of white chocolate into the European Union illustrate a number of the concepts behind management of duties and measure components.

The full duty applicable to the import of white chocolate (1704903000) is as follows:

9.10 % + 45.10 EUR / 100 kg MAX 18.90 % + 16.50 EUR / 100 kg

This is a complex duty that is a pair of compound duties joined together by a MAX conjunction.

Duty expressions

Usable duty expressions

The duty expression indicates to the border system how to treat the values that are attributed to the other columns in the measure_component table.

Duty expressions cannot be changed, appended or deleted.

Duty expression sequencing & duality

How to group duty expressions

Requirement Use duty expression …
To add an ad valorem duty or specific duties … 01, 04, 19 and 20
To add an ad valorem duty or specific duties after a MAX clause 17 and 35
To add an ad valorem duty or specific duties after a MIN clause 15
To add a Meursing placeholder … 12, 14, 21, 25, 27 and 29
To add a supplementary unit 99

It is not up to you to choose which of the grouped duty expressions to use:

  • use 01 in all occasions for an ad valorem duty or specific duties …
  • unless there is a compound duty, in which case use 01 followed by 04

Previously we described the duty expressions 01, 04 and 19 (and 20) as being functionally identical, however, they display differently – this is solely down to the other values in the table row.

For standard duty expressions (excluding Meursing and supplementary units):for a set monetary unit there is a specific duty. If there is no value, then it is an ad valorem duty. This is why the duty expression description is “% or amount”.

For a specified monetary unit, there must be a measurement unit. For a specified measurement unit, you can also apply a qualifier. It is used to specify measurement units such as “kg net weight” or “kg gross weight”.

Measurement unit qualifiers can only be applied to some measurement units.

The measurement units that can be used

NAR (number of articles) is commonly used, but can be referred to as:

  • “p/s” for items in French or German, or
  • “heads” in reference to livestock

Managing duties

When a trader declares goods for import, they usually have to declare the weight of their goods in kilograms (KGM), but goods may also be measured in:

  • “number of items” for livestock
  • volume in litres for liquid items
  • “number of pairs” for socks

The trader is asked to complete Box 41 on the SAD form (shown below) – the trader must enter the numeric value and the supplementary unit, for example 247 items (NAR) or 15,000 litres of pure alcohol (LPA).

How to create supplementary units in the tariff

Create a measure of type 109 (for imports and exports) or 110 (used for imports only).

Use 109 typically as this is a x-purpose measure type used for both imports and exports.

Type 110 is used solely for imports.

Create a measure component, using 99 as the duty expression ID and set the measurement unit and a measurement qualifier unit if needed.

Measure component data structures

Measure condition components

Measure conditions may be attached to conditions instead of to measures themselves.

Measure condition components measure components must not be applied together.

Measure condition components differ from measure components only in that they are attached to the condition and not the measure itself.

If the flag is set “not permitted” then no measure component or measure condition component must exist. Measure components and measure condition components are mutually exclusive. A measure can have either components or condition components (if the ‘duty expression’ flag is ‘mandatory’ or ‘optional’) but not both. Required This describes the fact that measures of certain types MUST have components (duties) assigned to them, whereas others must not. Please note the sub-clause also – if the value of the field “Component applicable” is set to 1 (Mandatory) on a measure type, then when the measure is created, there must be either measure components or measure condition components assigned to the measure, but not both.
