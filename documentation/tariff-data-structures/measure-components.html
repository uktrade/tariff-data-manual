Measure components are duties that are applied to tariff measures, not all measures have duties associated with them.

Introduction to measure components

  • Measure components basically equate to duties applied to tariff measures.
  • Not all tariff measures have duties associated with them – for instance measures of type 103, 142 or 143 (third country duty, tariff preference and preferential tariff rate quota) require duties, but for others, e.g. 277 Import prohibition, measure components are not permitted.
  • The screen grab to the right shows the duties / measure component column on the UK Trade Tariff service.

Types of duty

Broadly speaking, there are three kinds of duties that can be applied to a measure:

  • Ad valorem duty, for example 0.00% or 3.60% (a percentage additional duty applied to the import duty of the goods)
  • Specific duty, for example 4.200 EUR / KGM or 110.00 EUR / LTR (an import duty that is dependent on the weight / volume of goods being imported and not just the import price)
  • Compound duty – a combination of the above ad valorem and compound types, e.g. 3.60% + 110 EUR / LTR

In addition, there are ‘conjunctions’ which join multiple ‘clauses’ within the duty sentence, as follows:

Min (down to a minimum of …) Max (up to a maximum of …)

Fourthly, there are duty placeholders that are used to manage Meursing code inserts for complex agri-foods – though this is not a policy that the UK intends to take forward after the implementation period, it is worth bearing this in mind – this is explained in a separate document “Meursing codes”

Finally, there are is a concept called “Supplementary units” which is dealt with in the ”Measurements” document but referenced here.

Introduction to duty expressions

  • The table lists all of the possible duty expressions that can be used.
  • The duty expression indicates to the border system how to treat the values that are attributed to the other columns in the measure_component table.
  • Although there are 32 listed, very few of these are used on a regular basis. The ones that are used regularly are highlighted in yellow.
  • Those highlighted in green are massively prevalent right now in the EU tariff, but are expected to fall away in the UK tariff after the implementation period (Meursing codes).
  • The numbers of the duty expression IDs are essentially arbitrary, but they imbue meaning that is shared across Taric-based systems.
  • They are immutable – we cannot change or append to these and we cannot delete any unless we want to cause significant heartache for the CDS team in HMRC.

Duty expression sequencing & duality

Grouping duty expressions

Requirement Use duty expression …
To add an ad valorem duty or specific duties … 01, 04, 19 and 20
To add an ad valorem duty or specific duties after a MAX clause 17 and 35
To add an ad valorem duty or specific duties after a MIN clause 15
To add a Meursing placeholder … 12, 14, 21, 25, 27 and 29
To add a supplementary unit 99

And that’s pretty much it. It is not up to you to choose which of the grouped duty expressions to use: * use 01 in all occasions for an ad valorem duty or specific duties … * unless there is a compound duty, in which case use 01 followed by 04 * etc.

White chocolate’s duty string contains four elements, as per the measure components:

  • 9.10 % - a simple ad valorem (duty expression 01)
  • 45.10 EUR / 100 kg – a specific duty (duty expression 04)
  • MAX 18.90 % - a simple ad valorem (again), introduced by a Max conjunction (duty expression 17)
  • 16.50 EUR / 100 kg – a second specific duty (duty expression 19)

On the previous slide, we describe the duty expressions 01, 04 and 19 (and 20) as being functionally identical, however, they display differently – this is solely down to the other values in the table row.

Simply speaking, follow these rules of thumb for standard duty expressions (excluding Meursing and supplementary units):

  • if there is a value set in the monetary unit code column, then we are looking at a specific duty; if there is no value in this column then this is an ad valorem duty. This is why the duty expression description is ”% or amount”
  • the monetary unit will always be EUR (Euros)
  • if the monetary unit is specified, then the measurement unit must also be specified: you cannot have a specific duty that requires the importer to pay €x per …
  • if the measurement unit is specified, you may also apply a qualifier – this is not mandated and is in fact quite uncommon, but is used to specify measurement units such as “kilogramme net weight” or ”kilogramme gross weight” etc.
  • The measurement unit qualifiers can only be applied to certain pre-mandated measurement units.

About measurement units

  • The tables above identify the measurement units that can be used.
  • As with many of the Taric objects, there is a lot of historical content that has not been used for some time.
  • The yellow highlighted cells are those measurement units that have been used in the past 10 years
  • It may be appropriate to hide the less recently used measurement units from tariff managers.
  • The IDs of these units are not always especially self-explanatory
  • Pay special attention to NAR, which is very commonly used. Presumably, this is a German abbreviation (e.g. Nummer Artikeln), but it means number of items. To confuse even more, on the Trade Tariff Service and on the EU Taric Consultation, this is referred to as “p/s” (which stands for pièces / Stücke (items in FR / DE). Finally number of items is often referred to as “heads”, when referencing livestock, e.g. cattle especially in quotas. You have been warned.

Supplementary units

On most occasions, when a trader is declaring their goods for import, they are mandated to declare the weight of their goods in kilogrammes (KGM).

However, there are times where this is not the measurement required – common measurement types might be “number of items” for livestock, volume in litres for liquid items, “number of pairs” for socks etc. (really).

On such an occasion the trader is asked to complete box 41 on the SAD form (below) – the trader must enter the numeric value and the supplementary unit, e.g. 247 items (NAR) or 15,000 litres of pure alcohol (LPA).

The supplementary units are the same measurement units as described previously in this document. They are set up in the Tariff as follows – they are measures like any other:

Step 1 – create a measure of type 109 or 110

  • use 109 typically as this is a x-purpose measure type used for both imports and exports
  • type 110 is used solely for imports
  • there is also a type 111 for exports: it is never used

Step 2 – create a measure component

  • use 99 as the duty expression ID
  • set the measurement unit code field to whichever measurement unit is to be used
  • you may also set a measurement qualifier unit, however this is extremely rare.

Measure component data structures

Measure condition components

Measure conditions may be attached to conditions instead of to measures themselves. It is important that one does not double up and apply measure condition components AND measure components - this will break the business processing rules of Taric and fail to load on CDS. The data structures concerning measure condition components are almost identical to those for measure components – the only real difference being that they are attached to the condition and not the measure itself.

ME40 If the flag “duty expression” on measure type is “mandatory” then at least one measure component or measure condition component record must be specified. If the flag is set “not permitted” then no measure component or measure condition component must exist. Measure components and measure condition components are mutually exclusive. A measure can have either components or condition components (if the ‘duty expression’ flag is ‘mandatory’ or ‘optional’) but not both. Required This describes the fact that measures of certain types MUST have components (duties) assigned to them, whereas others must not.
