Introduction to footnotes

Structure of footnotes & footnote types

  • Much like certificates and additional codes, footnotes are made up of a type and then a specific ID, which, when concatenated, form the unique ID of the footnote.
  • For example, the footnote CD491 is constructed of a type (CD) and a specific footnote ID (491) – the entire footnote is known as CD491.
  • The numeric IDs, running from 001 to 999 can be reused across multiple footnote type IDs
  • In actual fact, footnote IDs may be either 3 or 5 digits long according to the XSD – all footnotes to date have been 3 digits long and while there is no clear reason to change, as footnotes are not so voluminous that the numbers are running out in many of the current database’s footnote type, there is benefit in considering a move to 5 digits (see below)
  • Footnote type IDs are either 2 alphabetical characters (those set by the EU) or 2 numeric characters (set as national footnotes by the UK). As with the footnote ID, there is additional redundancy here – the data schema gives an opportunity to use 3 digit footnote ID types – there should be no reason to adopt these.

Footnote type application codes

Application code Description Applies to … Notes
1 CN nomenclature Commodity codes •These two application codes are both used for footnote types that are to be linked to commodity codes (goods nomenclatures). •The difference is that application code 1 limits the use of the footnote such that it can only be applied to commodity codes where the first 8 digits are significant (i.e. ends in “00”). •The footnote types associated with application code 2 can be assigned to any commodity code (the Taric nomenclature label references the full use of all 10-digits in Taric). •There is no clear reason to ever create a footnote that can only be applied to commodity codes with 8 significant digits.
2 TARIC nomenclature Commodity codes
3 Export refund nomenclature ERNs •ERNs are historical - ignore
4 Wine reference nomenclature n/a •Wine references are historical - ignore
5 Additional codes Additional codes •While it is feasible that footnotes could be assigned to additional codes, there is no value, as there is no public way to display them
6 CN measures Measures •These two types share a similarity with the description of the nomenclature application codes above. •While they are both applied to measures, only footnotes of application code type ‘7’ can be applied to any measures, regardless of the tier in the code hierarchy to which the measure is assigned •There is no clear reason to ever create a footnote that can only be applied to measures with 8-digit commodity codes.
7 Other measures Measures
8 Meursing Heading Meursing headings •Meursing codes are / will be historical - ignore
9 Dynamic footnote n/a •While it is tempting to look at the dynamic footnote type and imagine that it can be applied to any of the different data objects with impunity, and that may have been the intention initially, it is pointed to note that the EU has only ever created one type with this application code (TP – Dynamic footnote) and the last time is was used was in 2015 – it is an answer to a question that has not been posed, and should not be used.

Proposed new footnote types

  • Footnotes are the last thing to be considered in a migration, as they are informational but not operational – the tariff will still function if the footnotes are omitted.
  • The only exception to this rule has been the ECJU, which has been very keen to create footnotes that explain the requirement with regard to export controlled goods (such as dual use, goods for torture or sanctioned countries)
  • Therefore, in migration, there will be very little attention paid to footnotes, unless a sea change in direction comes about during 2020.
    • No footnotes will be terminated
    • No associations of footnotes to commodity codes will be terminated
    • As EU measures are replaced by UK equivalents, in some cases the footnote assignment is not carried forward, though in like for like cases (e.g. a TAC agreement), the footnotes are carried forward like for like
    • Even where footnotes reference specific EU legislation, the decision was taken to retain the footnotes as-is – it has been a case of ‘bigger fish to fry’.
  • The key changes that the project team has proposed is to create three new footnote types for the benefit of the UK tariff & Jersey / Guernsey
FOOTNOTE TYPE ID DESCRIPTION APPLICATION CODE
FC Footnote to be applied to goods classification 2 (commodities)
FM Footnote to be applied to measures 7 (measures)
JG Footnote applicable specifically to Jersey / Guernsey 7 (measures)
  • Our proposal is to simplify the identification of footnotes, by in the future using only 2 footnote types for the majority of entities and to extend to 5-digits for footnote IDs.
  • The only exception could be in areas of specific interest, such as Dual Use Goods (which use the footnote type ID DU), though the preference should be for a clean slate – all footnotes use type FM for measures and type FC for commodities.

Key business rules - footnotes

# Rule Notes
FOT1 The type of the footnote must be unique. Required – High priority
FOT2 The footnote type cannot be deleted if it is used in a footnote. Required – Low priority Deletes are rarely required
FOT3 The start date must be less than or equal to the end date. Required – Medium priority End date should not be set
FO1 The referenced footnote type must exist. Required – High priority As the footnote type is selected from a dropdown, this should not go wrong
FO2 The combination footnote type and code must be unique. Required – High priority
FO3 The start date must be less than or equal to the end date. Required – High priority
FO4 At least one description record is mandatory. The start date of the first description period must be equal to the start date of the footnote. No two associated description periods may have the same start date. The start date must be less than or equal to the end date of the footnote. Required – High priority As a ‘type 2’ data object where the description can change over time, the footnote descriptions object cannot exist without at least 1 description record & equivalent period record that starts on the same day as the footnote record.
FO5 When a footnote is used in a measure the validity period of the footnote must span the validity period of the measure. Required – High priority End date for footnote is normally left open-ended.
FO6 When a footnote is used in a goods nomenclature the validity period of the footnote must span the validity period of the association with the goods nomenclature. Required – High priority End date for footnote is normally left open-ended.
